Was ist eine Klasse?



Was ist eine Klasse eigentlich?

Eine Klasse entspricht einem Konstruktionsplan für Objekte. Sie enthält die Beschreibung der Daten-Stuktur und des Verhaltens von Objekten, die von ihr erzeugt (instanziiert) werden können. (So ist z.B. Ihr Nachbar ein Objekt, also eine Instanz der Klasse „Mensch“ und Sie selbst sollte das auch sein.) Jedes Objekt hat dann die gleichen Attribute, jedoch verschiedene Attributswerte (Belegung der allgemeinen Attribute mit speziellen Werten).
Eine Klasse definiert nicht nur Attribute, sondern auch Methoden. In diesen werden die Verhaltensmöglichkeiten beschrieben, die für alle Objekte der Klasse gleich sind. (So sollte die Methode „atmen“ für jedes Objekt der Klasse Mensch möglich sein.)

Wie sieht so eine Klasse denn in Jumli aus?

Klassen werden im Jumli durch Rechtecke dargestellt, die in drei jeweils durch eine horizontale Linie getrennte Segmente unterteilt sind. Hat eine Klasse noch gar keine Attribute und Methoden, dann ist nicht viel von ihr zu sehen. Nur der Name und ein gelber Kasten erscheinen im Modell-Diagramm.

Im oberen Segment steht der Klassenname, der in diesem Fall 'Maedchen' lautet. Im mittleren Segment werden die Attribute namentlich aufgeführt. Hier können zusätzliche Angaben zu ihrem Typ, dem Initialwert, dem Zugriff (private, protected, public, unspecified) und den Eigenschaften (final bzw. static) notiert werden. Im unteren Segment werden die Methoden ebenfalls mit Namen, eventuell mögliche Parameter, deren Typ und Initialwert sowie Eigenschaften und Zugriff notiert.



Abstrakte Klassen:

Von abstrakten Klassen kann kein Objekt direkt erzeugt werden. Vielmehr müssen durch eine Vererbung die Eigenschaften der Klasse an die Unterklasse weitergegeben werden, um sie zu nutzen.



Interface Klassen:

Ein Interface (Schnittstelle) ist -wie eine abstrakte Klasse- eine Klasse, von der keine Objekte direkt erzeugt werden können. Zusätzlich gilt, dass ein Interface keine Attribute, sondern nur Methoden vorsieht, deren Ausgestaltung vollkommen den Unterklassen überlassen wird. Findet eine Vererbung statt, so ist man verpflichtet, alle im Interface deklarierten Methoden zu überschreiben, man muß also die Funktionalität der Methoden festlegen.

Man kann eine Interface als ein Versprechen der Unterklasse ansehen, das angibt, dass die Unterklasse alle Funktionalitäten des Interface bereitstellt.
