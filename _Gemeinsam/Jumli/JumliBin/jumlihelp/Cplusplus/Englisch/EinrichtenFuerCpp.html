C++ - Compiler Setup

In principle Jumli can use each compiler, we advise mingw. Mingw (minimalist GNU For Windows) is the Windows Port of the well-known GNU compiler GCC. The version 3.0.0-1 (MinGW-3.0.0-1.exe) can be downloaded e.g. under http://www.mingw.org/download.shtml. The installation uses the directory e.g. in c:\programme\mingw.

Subsequently, the subordinated bin directory has to add to the system path. For Win9x systems you can do this in the file autoexec.bat, there you should insert the following:

set PATH=c:\programme\mingw\bin:%PATH% the path must be naturally adapted to the respective listing!

For Windows NT systems you have to set the system path as this is explained also for Java compiler, only that the path to set must be e.g. c:\programme\mingw\bin. The explanations you can find here.

Additionally MSYS is needed. MSYS supplies further GNU Tools e.g. make. Download likewise under the address (MSYS-1.0.9.exe), specified above. The installation takes place either in the mingw directory or e.g. under c:\programme\msys. Afterwards, as described already above, the paths have to set,:

The bin directory must be taken up now to the system path FORWARDS mingw. Thus e.g..

set PATH=c:\programme\msys\bin:c:\programme\mingw\bin:%PATH%

Starting from version 1.3 Jumli uses makefiles for building the projects (a makefile is a kind of a control file, which contains information for compiling and linking. Descriptions in more detail find you simple in the internet, put into a search machine the terms „makefile“ and „introduction“).

To produce makefiles tmake (Download: http://www.trolltech.com/download/tmake.html ) is used. Unpack the zip archives e.g. after c:\programme\tmake and afterwards again add the bin directory to the path, this is according to the same pattern, as it is described already above 2 times. Additionally still the new environment variable must be set TMAKEPATH, dependent on the used compiler. For Mingw e.g. on c:\programme\tmake\lib\win32-g++.

wxWindows

Jumli can provide also wxWindows projects now. WxWindows ( http://www.wxwindows.org ) is a free system-independent class library, which is available for Windows, Linux and other Unix derivatives. Jumli itself was migrated with version 1.3 to wxWindows. After the Download and the installation into any directory you first have to compile wxWindows. This must take place with the same compiler, which is to be used afterwards in Jumli! If you use Mingw the following steps are necessary:

  • change into the installation directory

  • cd src/msw

  • make -f Makefile.g95

  • then the same prodedure for the contrib-libraries

  • change into the installation directory

  • cd contrib\src

  • make -f Makefile.g95

  • set the environment variable WXWIN to the installation directory

While create a project Jumli analyses the wxWindows documentation A condition therefor is the installed HTML documentatopn for wxWindows, that is available likewise on the wxWindows homepage. After the Download this must simply unpacked, at the best one into the already existing installation directory of wxwindows, thus e.g. c:\wxwindows\docs.

And now enjoy working with jumli.
