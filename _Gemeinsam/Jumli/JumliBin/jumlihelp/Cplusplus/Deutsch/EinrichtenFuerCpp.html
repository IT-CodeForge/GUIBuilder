C++ - Compiler Setup

Jumli kann prinzipiell jeden Compiler verwenden, wir empehlen mingw. Mingw (Minimalist GNU For Windows) ist die Windows-Portierung des allseits bekannten GNU-Compilers gcc. Die zum Erstellungszeitpunkt dieser Seite aktuellste Version 3.0.0-1 (MinGW-3.0.0-1.exe) kann z.B. unter http://www.mingw.org/download.shtml heruntergeladen werden. Die Installation erfolgt dann z.B. in c:\programme\mingw.

Anschließend muß das untergeordnete bin-Verzeichnis in den Systempfad aufgenommen werden. Für Win9x Systeme erfolgt diese Einstellung über die Datei autoexec.bat, dort sollte folgendes eingtragen werden:

set PATH=c:\programme\mingw\bin:%PATH%

Der Pfad muß natürlich an das jeweilige Verzeichnis angepasst werden!

Für Systeme ab Windows NT werden die Einstellungen in der Systemsteuerung vorgenommen.Dies funktioniert genau so, wie dies auch für das Java-Compiler erklärt wird, nur dass Sie nun den oben genannten Pfad eintragen müssen ( c:\programme\mingw\bin ). Die angesprochenen Erklärungen finden Sie hier.

Wer seine Pfade absolut nicht verändern will kann Jumli aus einer Dosbox heraus starten und dort die Einstellung lokal z.B. über eine Batch-Datei vornehmen.

Zusätzlich wird MSYS benötigt. MSYS liefert weitere GNU-Tools wie z.B. make. Download ebenfalls unter der oben genannten Adresse (MSYS-1.0.9.exe). Die Installation erfolgt entweder ins mingw-Verzeichnis oder z.B. unter c:\programme\msys. Danach müssen, wie bereits oben beschrieben, die Pfade erweitert, also gesetzt, werden:

Das bin-Verzeichnis muss nun in den Systempfad VOR mingw aufgenommen werden. Also z.B.

set PATH=c:\programme\msys\bin:c:\programme\mingw\bin:%PATH%

Ab Version 1.3 verwendet Jumli Makefiles zum Bauen der Projekte (Ein Makefile ist eine Art Steuerdatei, die Informationen für das Compilieren und Linken beinhaltet. Ausführlichere Beschreibungen finden Sie im Internet, einfach in eine Suchmaschine die Begriffe „Makefile“ und „Einführung“ eingeben).

Für die Makefile-Erzeugung kommt tmake ( Download: http://www.trolltech.com/download/tmake.html ) zum Einsatz. Das Zip-Archiv z.B. nach c:\programme\tmake entpacken und anschließend wieder das bin-Verzeichnis in den Pfad aufnehmen, dies erfolgt wieder nach dem gleichen Schema, wie es bereits oben 2 Mal beschrieben wird.. Zusätzlich muß noch die Umgebungsvariable TMAKEPATH gesetzt werden, und zwar abhängig vom verwendeten Compiler. Für Mingw z.B. auf c:\programme\tmake\lib\win32-g++. Dies geschieht ähnlich wie das Setzen des Pfades, nur dass Sie dieses Mal im Dialog der Umgebungsvariablen den Button „Neu“ anstatt „Bearbeiten“ wählen müssen. Als Namen geben Sie hier TMAKEPATH an, der Wert der Variable ist dann der eben beschriebene Pfad.

wxWidgets

Jumli kann jetzt auch wxWidgets-Projekte erstellen. wxWidgets ( http://www.wxWidgets.org ) ist eine freie systemunabhängige Oberflächenbibliothek, die z.B. für Windows, Linux und andere Unix-Derivate verfügbar ist. Jumli selbst wurde mit Version 1.3 nach wxWidgets portiert. Nach dem Download und der Installation in ein beliebiges Verzeichnis muß wxWidgets zuerst kompiliert werden. Dies muß mit demselben Compiler erfolgen, der nachher in Jumli verwendet werden soll! Für den hier verwendeten Mingw sind folgende Schritte erforderlich:

  • ins Installationsverzeichnis wechseln

  • cd src/msw

  • make -f Makefile.g95

  • dann dasselbe nochmal für die contrib-Bibliotheken

  • ins Installationsverzeichnis wechseln

  • cd contrib\src

  • make -f Makefile.g95

  • und danach noch die Umgebungsvariable WXWIN auf das Installationsverzeichnis setzen

Beim Anlegen eines Projekts parst Jumli die wxWidgets-Hilfe (HTML). Voraussetzung hierüfr ist die installierte html-Hilfe für wxWidgets, die ebenfalls auf der wxWidgets-Homepage erhältlich ist. Diese muss nach dem Download einfach entpackt werden, am Besten in das bereits vorhandene Installationsverzeichnis von wxWidgets, also z.B. c:\wxWidgets\docs.

Und nun viel Spaß mit Jumli.
