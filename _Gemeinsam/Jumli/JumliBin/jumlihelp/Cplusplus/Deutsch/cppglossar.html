Wichtige Begriffe für Jumli und C++

Compiler

C++ ist eine sogenannte Hochsprache. Sie wurde entwickelt, um dem Menschen das Programmieren zu vereinfachen. Sie arbeitet auf viel höherem Abstraktionsniveau als die Maschinensprache, die Ihr PC ausführen kann. Der Compiler ist nun ein Programm, das Ihr C++-Programm "versteht" und daraus die richtigen Maschinenbefehle für Ihren Rechner erzeugen kann, damit der nachher auch das tut, was im C++-Programm beschrieben ist.

Ein C++-Programm muß nach dem Compilieren auch noch gelinkt werden. Was das bedeutet, lesen Sie hier.

Übrigens: Wenn es nicht so läuft, wie es soll, dann liegt es meistens an Ihrem Programm und nicht am Compiler. Oder, wie die Informatiker sagen: "Meistens sitzt das Problem davor."

Debugger

Auch dem besten Programmierer passieren Fehler bei der Arbeit. Das ist ganz normal. Ein Programm, das auf Anhieb fehlerfrei läuft, das gibt es nicht - solange es in der Funktionalität über das einfache "Hallo Welt!" hinausgeht. Programmfehler sind nicht schlimm. Schlimm ist es, wenn man sie nicht findet. Doch dafür gibt es Unterstützung:

Der Debugger ist ein Programm, das dem Programmierer hilft, Fehler in seinem Programm zu finden. Mit Hilfe des Debuggers können Sie zum Beispiel Schritt für Schritt verfolgen, was beim Programmlauf passiert. Sie können sozusagen zusehen, was der Computer tut. Und Sie können sich jederzeit die Inhalte der Programmvariablen ansehen und bei Bedarf auch verändern. Und Sie können soganannte Breakpoints setzen, um das Programm an einer bestimmten Stelle anzuhalten, die Sie sich genauer ansehen möchten.

linken

"linken", das hat nichts mit links oder rechts zu tun, und reingelegt wird dabei auch niemand.

Der compilierte Code Ihres C++-Programms braucht in der Regel noch Funktionalität aus anderen Bibliotheken. Das kann (zumindest) die Laufzeitbibliothek des Compilers sein oder sogenannte Importbibliotheken, die den Zugriff auf Funktionen in DLLs (so können Sie zum Beispiel fast alle Funktionen von Ihres Betriebssystems nutzen) ermöglichen. Oder sogenannte statische Bibliotheken, die fertig compilierten Code enthalten und diesen für andere Programme bereitstellen. Statische Bibliotheken haben genau wie die Importbibliotheken üblicherweise die Endung ".lib"

Die für ein Programm benötigten Funktionen muß also aus vielen Quellen zusammengefügt werden. Diesen Vorgang nennt man "linken", und das Programm, welches dies erledigt, ist der "Linker".

Oft sind Compiler und Linker im selben Programm vereint, und über Kommandozeilenparameter kann gesteuert werden, ob compiliert, gelinkt oder beides gemacht werden soll.

int main()

Jedes C++-Programm braucht einen Einstiegspunkt, damit der Compiler weiß, an welcher Stelle es losgeht. Dies ist in C++ die Funktion int main(), die wie folgt deklariert ist:

int main();

oder 

int main( int argc, char ** argv );

Im Unterschied zu Java ist die main()-Funktion in C++ eine freistehende Funktion und nicht Methode einer "Haupt"-Klasse.
Das liegt zum einen daran, daß C++ in dieser Hinsicht kompatibel zu C sein will, und zum anderen daran, daß C++ mehrere Programmierparadigmen unterstützt, während Java nur die objektorientierte Programmierung unterstützt.

(Wie sagt man so schön: "Wenn das einzige Werkzeug ein Hammer ist, dann sehen alle Probleme wie Nägel aus.")

In C++ können Sie objektorientiert programmieren, Sie müssen aber nicht.

Und jetzt kommen Sie bitte näher... Ja, so ist es gut... Denn jetzt kommt etwas sehr Wichtiges:

    Die main-Funktion hat den Rückgabewert int und nicht void. Das ist im C++-Standard so definiert. Es gibt zwar einige Compiler, die auch eine Deklaration mit void durchgehen lassen, aber das ist nicht C++. Sie wissen das jetzt, und deshalb werden Sie immer int main() schreiben und niemals void main(). Und wenn Sie irgendwo void main() sehen, dann wissen Sie es besser: Derjenige konnte nicht richtig C++.

    Und wenn Sie gar keinen Rückgabewert brauchen? Dann schreiben Sie am Ende der main()-Funktion einfach 

    return 0;

    Übrigens: Wenn Sie das weglassen, tut der Compiler so, als ob sie es hingeschrieben hätten. Es ist also (Ausnahme für die main()-Funktion!) kein Fehler, das return wegzulassen, obwohl der Rückgabetyp von main() int ist.

Es ist üblich, das ordnungsgemäße Beenden eines Programms mit dem Rückgabewert 0 anzuzeigen und Fehler mit Rückgabewerten ungleich 0 anzuzeigen. Damit können andere Programme, die Ihr Programm aufrufen, auf Fehler entsprechend reagieren.

Standard ISO/IEC 14882

Hinter dieser kryptischen Bezeichnung verbirgt sich das Dokument, in dem die Sprache C++ zu einem international anerkannten Standard geworden ist. Darin steht detailliert beschrieben, wie die Sprache aufgebaut ist.

Dieser Standard ist herstellerunabhängig - es kann also keine Firma einfach sagen, C++ ist auf einmal etwas anderes.
