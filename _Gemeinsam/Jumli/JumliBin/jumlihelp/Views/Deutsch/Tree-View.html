Das Tree-Fenster



Das Tree-Fenster besteht aus vier verschiedenen Registern. Diese sind das Projekt-Register, das Dateien-Register, das SDK-Register und das Info-Register. Im Weiteren werden diese näher erläutert.



Das Projekt:

Das Projekt-Register ist in drei verschiedene Unterpunkte eingeteilt.


Der erste Bereich enthält die Use-Case-Packages. Hier werden alle Use-Cases aufgelistet.

Der zweite Bereich enthält die restlichen Packages. Hier werden alle Diagramme und Klassen mit ihren Attributen und Methoden aufgelistet. Hierzu gehören auch Assoziationen und Vererbungen.
Die Diagramme im Projekt sind nur verschiedene Sichten auf das Model. Sie können also mehrere Diagramme in Ihrem Package haben. Löschen Sie ein markiertes Diagramm oder Model-Element per DEL-Taste aus dem Fenster, so wird nicht das eigentliche Element gelöscht. Dies kann nur über den jeweiligen Dialog des Model-Elementes geschehen.

Im dritten Bereich befinden sich die externen Packages und Klassen des SDK, sofern Sie Klassen aus dem SDK im UML-Modell eingefügt haben.

Die Dateien:

Das Dateien-Register enthält alle Dateien, die zu Ihrem Projekt gehören.


Der erste Bereich enthält die Use-Case-Packages. Hier werden alle Use-Cases aufgelistet.

Der zweite Bereich enthält die restlichen Packages. Hier werden alle Diagramme und Klassen mit ihren Attributen und Methoden aufgelistet. Hierzu gehören auch Assoziationen und Vererbungen.
Die Diagramme im Projekt sind nur verschiedene Sichten auf das Model. Sie können also mehrere Diagramme in Ihrem Package haben. Löschen Sie ein markiertes Diagramm oder Model-Element per DEL-Taste aus dem Fenster, so wird nicht das eigentliche Element gelöscht. Dies kann nur über den jeweiligen Dialog des Model-Elementes geschehen.

Im dritten Bereich befinden sich die externen Packages und Klassen des SDK, sofern Sie Klassen aus dem SDK im UML-Modell eingefügt haben.

Im vierten Bereich sind die generierten Files. Darunter versteht man den Java-Code der Klassen, die generiert wurden.

Im letzten Bereich befinden sich die sonstigen Files. Dies sind Java-Files, die z.B. aus anderen Packages oder Projekten hinzugefügt wurden, die aber noch nicht zum eigenen Projekt zählen. Diese Files können Sie über Code->Sourcecode parsen in ihr UML-Modell bringen. Erst dadurch können Sie die Datei compilieren. Wichtig ist, dass Jumli nur .java - Dateien unterstützt. .jar - Dateien können zwar ins Jumli importiert und im Edit-Fenster angezeigt werden, sie können aber nicht über das Parsen zum Projekt hinzugefügt werden. Unter den sonstigen Files finden Sie auch HTML-Dokumente, die Sie ins Projekt importiert haben.

Das SDK:

In diesem Register sehen Sie die Packages, Klassen und die Hilfe des SDKs, das Sie installiert haben. Mehr dazu finden Sie hier.

Durch Doppelklick auf einen Eintrag erhalten Sie die Dokumentation des SDK, sofern Jumli diese finden konnte.
Zusätzlich besteht die Möglichkeit, direkt aus dem Register Klassen in das eigene UML-Model einzubauen. Dazu müssen Sie diese nur anklicken und mit der Maus in den Jumli-View ziehen. Diese Klassen können nicht generiert werden, aber sie werden im Projekt-Register angezeigt und können im Model durch Vererbung verwendet werden.

Die Info:

Im Info-Register finden Sie die Hilfe-Seiten, in denen Sie sich im Moment befinden. Das Info-Register ist unterteilt in zwei Bereiche:

  • Jumli-Samples: Hier finden Sie Beispiel-Projekte, die das Arbeiten mit Jumli aufzeigen sollen. Sie können die Beispiele durch Doppelklick starten.

  • Jumli-HTML-Info: Hier finden Sie eine HTML-Hilfe zu vielen Themen rund um Jumli.
