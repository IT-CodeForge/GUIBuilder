Die fünf Fenster von Jumli



Rechtliche Hinweise



Überblick über alle Fenster



1. Das Tree-Fenster

Auf der linken Seite der Oberfläche ist ein schmales Fenster, genannt 'Tree-Fenster' angesiedelt. Es dient dazu, die Übersicht im Gewirr der vielen Klassen, Methoden, Attributen, etc. zu behalten. Das Fenster zeigt in einem bestimmten Schema vielen Einzelheiten eines Projektes an.

2. Das Modell-Fenster

Das zentrale Fenster von Jumli. Hier ist es möglich, mit Hilfe von UML, der Unified Modelling Language, Programmgerüste grafisch abzubilden, Klassen, Beziehungen, Attribute, und vieles mehr zu definieren um sie später einfach in Java Sourcecode zu generieren.

3. Das Debug-Fenster

Im diesem Fenster werden beim Debuggen Informationen angezeigt. Beispielsweise sieht man dort, an welcher Stelle der Debugger gerade steht und wo Sie Ihre Breakpoints gesetzt haben.

4. Das Edit-Fenster

In diesem Fenster am unteren Ende der Oberfläche steht der Java-Sourcecode, welcher entweder vorher generiert, in das Fenster geladen oder auch von Ihnen dort eingefügt wurde. Hier ist nicht nur die Ansicht sondern auch das Editieren von Java-Sourcecode möglich.

5. Das Message-Fenster

Alle Informationen vom Jumli werden im Message-Fenster angezeigt. Beachten Sie dieses Fenster also bei allen Aktionen, die Sie mit Jumli ausführen.
