Der Methoden Dialog




    1. Geben Sie hier den Namen der Methode ein. Er darf keine Leerstelle und kein Sonderzeichen enthalten.

    2. Hier können Sie einen Stereotype für die Methode vergeben.

    3. Hier müssen Sie den Rückgabewert der Methode eingeben. Standardmäßig ist 'void' vorgeschlagen. Es sind aber auch 'int' oder sogar ganze Klassen als Rückgabewert möglich

    4. Throws sollen Ausnahmen bei fehlerhaften Eingaben abfangen. Wird beispielsweise einer Methode, die eigentlich einen int-Parameter hat, ein String-Parameter mitgegeben, würde das Programm nicht laufen. Durch Throws sollen solche Fehler abgefangen werden.

    5. Hier können Sie den Zugriff auf die Methode festlegen. Sie haben die Auswahl zwischen:

      private: Die Methode ist nur innerhalb der Klasse sichtbar und bekannt.
      protected: Die Methode kann von Klassen innerhalb des Packages und von abgeleiteten Klassen aufgerufen werden.
      public: Alle Objekte, die die zugehörige Klasse kennen, können die Methode aufrufen.
      internal: Diese Option steht nur in C# zu Verfügung.
      protected internal: Auch dieses Zugriffsrecht kann nur in C# gentzt werden.
      package: Die Methode ist nur im Package bekannt.



6. Hier können Sie die Eigenschaften einer Methode definieren. Es gibt die Auswahl zwischen:

    abstract: Diese Methode muss von einer abgeleiteten Klasse überschrieben werden.
    final: Die Methode kann von keiner anderen Klasse überschrieben werden ( nur in Java möglich ).
    static: Die Methode wird nicht für jedes Objekt einzeln angelegt, sondern existiert nur einmal für die ganze Klasse. Alle Objekte dieser Klasse haben Zugriff auf diese Methode,d.h. wird sie von einem verändert, erscheint sie für alle verändert. Der Zugriff ist auch möglich, wenn noch kein Objekt dieser Klasse existiert.
    native: Diese Eigenschaft kann nur in Java genutzt werden.
    synchronized: Eine Klasse kann von mehreren Threads "zeitgleich" verändert werden. Setzt man die Eigenschaft synchronized, so wird sichergestellt, dass nur ein Thread je Zeiteinheit die Klasse verändert. Dies ist nur in Java möglich. explicit: Diese Eigenschaft kann nicht in Java gennutzt werden.
    const: In C++ kann eine Methode const deklariert werden.
    virtual:Auch diese Eigenschaft kann man nicht in Java nutzen.
    implicit, new, override, ist delegate: Diese Eigenschaften sind C# vorbehalten.

7. Die Parameter der Methode werden hier aufgelistet.

8. Mit NEU kommen Sie in den Parameter Dialog.

9. Mit ENTF können Sie den markierten Parameter entfernen, dann wird der Parameter nach einer nochmaligen Abfrage gelöscht.
