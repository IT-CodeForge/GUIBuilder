Association Dialog





1. Enter the name of the association here.

2. Here you can enter a rolename.

3. This field contains the name of the initial class. It cannot be edited in this dialog.

4. Here you can determine the multiplicity of the association on the side of the initial class. There are some standard-values suggested.

5. This field contains the name of the end class. It cannot be edited in this dialog.

6. Here you can determine the multiplicity of the association on the side of the end class. There are some standard-values suggested.

7. If you like to change the type of the association into a aggregation or a composition you can make your choice here.

8. Here you can determine the visibility of the association. You can choose between:

    private: set the visibility to private
    protected: a protected object will be generated
    public: a public object will be generated
    internal: an internal object will be generated ( C# only )
    unspecified: an unspecified object will be generated

9. If you want Jumli to automatically generate Get and Set methods for this association you can select this options here.
If you select "Array", an array will be generated in the initial class ( impossible for C++ ). The Array is a kind of table in which multiple objects of the same data type can be inserted (in this case it means objects having the type of the end class).
In contrast to an array, a vector can also contain objects of different data types. ( Java only ).

10. Exit the dialog and save your changes.

11. Exit the dialog without saving your changes by clicking 'Cancel'.

12. This button lets you delete the current association. After pressing 'Delete' you have to confirm with OK. Now you will be asked if you are sure that you are sure, and if you do so, the association will be deleted.

13. Click here to open the docu-dialog.

14. This button leads you to the dialog help, where you are right now.
