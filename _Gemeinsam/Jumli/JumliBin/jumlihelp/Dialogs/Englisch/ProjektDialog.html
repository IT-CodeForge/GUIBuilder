Projekt Einstellungen






    1. Präfix für Get-Methoden von Attributen und Aggregationen. Standard für Java und C# ist get, für C++ Get. Beispiel: Der Attributname ist „Name“, dann generiert Jumli die Methode „getName()“.

    2. Wie get.

    3. Attribute werden mit dem angegebenen Präfix generiert.

    4. Nur C++: Bei Methoden ohne Parameter soll „void“ generiert werden.

    5. Die Einstellungen werden für alle neuen Jumli-Projekte als Standard übernommen.






    1. Nur Java: List mit den eingebundenen Jar-Files.

    2. Startet den File-Dialog, um ein Jar-File hinzuzufügen.

    3. Entfernt das selektierte Jar-File aus dem Projekt.

    4. Die Einstellungen werden für alle neuen Jumli-Projekte als Standard übernommen.






    1. Falls ein (Server-)Verzeichnis angegeben wird, ermöglicht Jumli eine Multi-User Projektentwicklung ohne SCM-System. Auf dem Verzeichnis muß eine Referenz des Projekt gespeichert sein. Jeder Entwickler arbeitet mit einer lokalen Kopie, indem er Sourcecode-Dateien sperrt, ändert und die Änderungen freigibt. (Momentan nur für Änderungen im Sourcecode und nicht für das Modell möglich.)

    2. Hier kann das Verzeichnis für das Referenzprojekt eingestellt werden.

    3. Der User-Name, unter dem ein Entwickler Dateien sperrt. In den meisten Fällen muß kein Name eingegeben werden, da Jumli zuerst versucht, den Namen direkt aus dem System zu ermitteln.

    4. Die Einstellungen werden für alle neuen Jumli-Projekte als Standard übernommen.
