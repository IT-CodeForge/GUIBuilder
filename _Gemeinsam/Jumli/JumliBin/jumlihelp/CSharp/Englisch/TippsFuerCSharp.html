tips



There's a problem if both versions of the SDK were installed on the same system. Then it is possible that Jumli can't find the actual compiler directory. If this happens the best way to solve this problem is to set the path manually. Therefore you have to open the SDK options dialog in Jumli.
Now there's the question where the documentation and the compiler is to be found. The best way to figure this out is to let the system search. There's a file named 'starthere.htm' in the documentation directory, e.g 'C:\Programme\Microsoft.NET\FrameworkSDK\''. This could be the searched directory – but it doesn't have to be the rigth one.

Now you have to find the compiler directory, the compiler is named 'CSC.exe'.
