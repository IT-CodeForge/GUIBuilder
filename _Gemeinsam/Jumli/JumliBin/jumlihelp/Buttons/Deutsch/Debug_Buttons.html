Debug Buttons


Button run

Durch den Button run startet der Debugger das Programm. Er läuft entweder bis zum ersten Breakpoint, bis zum ersten Fehler oder bis ans Ende des Programms und verläßt dann den Debug-Modus.



Button exit

Mit diesem Button verlassen Sie den Debugger und kommen zurück zu Ihrem normalen Edit-Fenster. Erst wenn im Message-Fenster 'Debug beendet!' angezeigt wird, können Sie weiterarbeiten.



Button cont

Kommt der Debugger an einen Breakpoint, so bleibt er an dieser Stelle stehen. Wollen Sie, dass Ihr Programm weiter durchlaufen wird, so klicken Sie auf den Button cont. Der Debugger läuft dann bis zum nächsten Breakpoint oder, falls kein weiterer Breakpoint gesetzt ist, bis ans Ende Ihres Programmes.



Button stop at

Der Button stop at bewirkt, dass ein sogenannter Breakpoint (dt. Brechpunkt) gesetzt wird. Dieser Breakpoint wird in der Zeile gesetzt, an der Ihr Cursor im Fenster blinkt. Wird der Debugger gestartet, so bleibt er bei der Ausführung des Programmes an dieser Stelle stehen. Dadurch ist es möglich, den Inhalt verschiedener Variablen zu überprüfen, um zu sehen, ob das Programm richtig arbeitet und ob keine Logik-Fehler im Programm enthalten sind.



Button clear

Über diesen Button können Sie gesetzte Breakpoints wieder löschen. Dazu müssen Sie in die jeweilige Zeile klicken bzw. diese markieren und dann den Button drücken.



Button step

Dieser Button führt nach einem Breakpoint genau eine weitere Anweisung aus und unterbricht anschließend die Ausführung.



Button next

Wenn Sie im Debug-Modus zum nächsten Java-Befehl springen wollen, dann können Sie dies mit Hilfe des next-Buttons.



Button step up

Mit diesem Button können Sie aus einem verschachtelten Befehl zurück in die aufrufende Methode springen.



Button dump

Sie können durch den dump-Button den Inhalt eines Objektes überprüfen. Dadurch sehen Sie, ob Ihr Programm richtig arbeitet oder falsche Werte erzeugt. Dazu müssen Sie das Objekt vorher markieren.



Button command

Der Button command öffnet ein Dialog-Fenster, in dem Sie zwischen verschiedene Kommandos wählen können. Wenn Sie das gewünschte Kommando in der oberen Zeile eingeben, wird es von Jumli ausgeführt. Über diesen Button können Sie verschiedene Kommandos auswählen, die aus Gründen der Übersichtlichkeit nicht als Button im Debug-Modus auftauchen.
