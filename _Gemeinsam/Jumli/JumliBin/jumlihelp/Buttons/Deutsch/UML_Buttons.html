UML Buttons


Die UML-Buttons sind nur im Modell-Fenster aktiv. Bei diesen Buttons ist es wichtig, dass Sie nicht nur den Button anklicken müssen, um ein UML-Element zu erzeugen, sondern Sie müssen anschließend die Stelle im Modell-Fenster anklicken, an der das ausgewählte Element im Fenster erscheinen soll. Tun Sie dies nicht, so kommen Sie auch nicht in den jeweiligen Dialog. Es wird kein UML-Element erzeugt.



Button Package

Über den Button Package kommen Sie in einen Dialog, in dem Sie die Eigenschaften eines Package definieren können. Näheres über die Einstellungen erfahren Sie im Package Dialog.



Button Diagramm

Mit dem Diagramm-Button können Sie im selben Projekt mehrere Diagramme erstellen. Verändern Sie eine Klasse in einem Diagramm, so ist sie automatisch auch in allen anderen Diagrammen verändert worden. Ein weiteres Diagramm ist also nur eine veränderte Sicht auf die bereits bestehenden Klassen und dient der Übersichtlichkeit. Um die Einstellungen eines Diagramms festzulegen, gehen Sie auf die Seite des Diagramm Dialoges.



Button Klasse

Wenn Sie eine Klasse erzeugen wollen, müssen Sie diesen Button drücken. Dadurch öffnet sich der Klassen Dialog. Dieser ist schon etwas komplexer. Wenn Sie genaueres wissen möchten, finden Sie dies im Klassen Dialog.



Button Assoziation

Bei einer Assoziation ist es wichtig, von einer Klasse zur anderen einen Pfeil zu ziehen, indem Sie die Maustaste gedrückt halten und sie erst über der zweiten Klasse wieder loslassen. Bei der Assoziation gibt es ebenso wie bei der Klasse verschiedene Einstellungen, wie z.B. Kardinalitäten. Um in den Assoziationen Dialog zu gelangen, müssen Sie auf die Assoziation doppelklicken.



Button Vererbung

Eine Vererbung wird ebenso wie eine Assoziation erzeugt. Mit einem Doppelklick auf die Vererbung kommen Sie in den Vererbungen Dialog, in dem Sie die Vererbung bearbeiten können.



Button Verantwortungsbereich

Ein Verantwortungsbereich wird ebenso wie eine Assoziation erzeugt. Mit einem Doppelklick auf den Verantwortungsbereich kommen Sie in den Verantwortungsbereichs- Dialog, in dem Sie den Verantwortungsbereich bearbeiten können.



Button Dependency

Die Erzeugung einer Dependency geschieht analog zu Assoziation und Vererbung. Auch hier öffnet sich mit einem Doppelklick auf die Dependency der dazu passende Dependency Dialog zur Bearbeitung der Dependency.



Button Objekt

Ein Objekt kann nur im Sequenz- oder Kollaborationsdiagramm erzeugt werden. Dadurch kommen Sie in den Objekt-Dialog.



Button Objekt-Nachricht

Die Objekt-Nachricht wird ebenso wie z.B. die Assoziation erzeugt. Sie kann nur von einem Objekt zum anderen erzeugt werden. Deshalb ist der Button, ebenso wie der Objekt-Button, nur im Sequenz- und Kollaborationsdiagramm aktiv. Näheres zu den Einstellungen für die Objekt-Nachricht finden Sie im Objekt-Nachrichten Dialog.



Button Objekt-Link

Ebenso wie die Objekt-Nachricht wird auch der Objekt-Link erstellt. Dieser Button ist aber nur im Kollaborationsdiagramm aktiv. Bei der Erstellung gelangen Sie direkt in den Objekt-Link Dialog.



Button Use Case

Use Cases können Sie nur im Use Case Diagramm erzeugen. Die möglichen Einstellungen zum Use Case finden Sie im Use Case Dialog.



Button Actor

Auch der Actor kann nur im Use Case-Diagramm erzeugt werden. Nachdem Sie diesen Button geklickt haben, kommen Sie in den Actor Dialog.



Button System

Wie auch der Actor, kann auch das System kann nur im Use Case-Diagramm erzeugt werden. Nachdem Sie diesen Button geklickt haben, kommen Sie in den System Dialog.



Button Notiz

Mittels Notiz-Button lassen sich Notiz-Elemente im Modell-Fenster erzeugen, deren Aufgabe darin besteht, bestimmte UML-Elemente zu kommentieren. Auf diese Weise können unübersichtliche Modell-Diagramme für Dritte verständlicher gestaltet werden.



Button Notiz-Link

Der Notiz-Link-Button fügt eine Verbindung zwischen Notiz und kommentiertem UML-Element ein. Mit gedrückter Maustaste zeihen Sie den Notiz-Link von der Notiz zum UML-Element.



Button Generieren

Wenn Sie den Button Generieren betätigen, springt Jumli in das Edit-Fenster. Dort wird Ihre Klasse als Sourcecode angezeigt. Im Message-Fenster wird zusätzlich angezeigt, was Jumli getan hat. Sie haben jetzt also das UML-Model in Programm-Code 'übersetzt'.
