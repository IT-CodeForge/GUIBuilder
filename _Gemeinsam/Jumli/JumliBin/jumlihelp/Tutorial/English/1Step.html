Well, I think we can start now. Don't be afraid, everything we'll do is really easy. To start, klick on "Datei" (in the upper-left corner of the screen). No need to get frightened if a grey list with many strange items appears. That's the way it sould go, it's called "Pop-up-Menu". It should look like this:

Eine neue Datei
If you move around the mouse the blue beam will move, too. You can select any menuitem by clicking on this item. That's all.
Click on "Neu", that will create a new project. The next menu should appear. It's the window for finding out, what kind of project our work will be.

... to the top    ...no prob, go ahead!


Well, what's your sex?

First of all Jumli wants to know, what the name of our project will be. In our exapmle you should enter "MyProject". That's the name of the project, not the name of following classes.
But you shurely noticed the two little caskets. There is "Applet" and "Application".
  • Applet: Definition...
  • Application: Definition...

To proceed you have to decide what your project is gonna be. This time we choos "Application". Then click on "OK" and we can go on.

... to the top    ...no prob, go ahead!


Where to save?

Next thing Jumli wants to know is where it should save the new project. Imagine Jumli would save the files where the sun never shines. what a jumble. The best thing you can do is to create a new folder called "JumliProject" for example where you can save all your projects. After that you have to click on OK and that's it.

... to the top     ...no prob, go ahead!


Hey, look at that!

Amazing, in the Tree-View on the left site of the screen you can see the new project "MyProject". There's a yellow icon in the Tree-View.
Every created project includes a so called package. This is show, too (a green icon). To be able to see which diagramms, classes, etc. are included in this package, you have to click on the little plus-symbol. Now the Tree-View is opening one more level and you can see further details.
